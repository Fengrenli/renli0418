# 文件上传指南

## 支持的文件类型

- **3D模型文件**: .glb, .gltf, .obj, .fbx, .ifc, .nwd, .dwg, .skp
- **文档文件**: .pdf, .doc, .docx, .xls, .xlsx
- **图片文件**: .jpg, .jpeg, .png, .webp
- **视频文件**: .mp4, .mov, .avi
- **其他文件**: .rvt

## 文件名规范

为了避免乱码问题，请遵循以下文件名规范：

1. **使用英文或数字**：优先使用英文字母和数字命名文件
2. **避免特殊字符**：不要使用以下字符：`\ / : * ? " < > |`
3. **保持简洁**：文件名长度建议不超过50个字符
4. **使用下划线替代空格**：例如 `project_model.glb` 而不是 `project model.glb`

## 推荐文件名格式

```
项目名称_文件类型_版本号.扩展名
```

**示例**：
- `xiaolongkan_dusseldorf_model_v1.glb`
- `houtang_dubai_plan_v2.pdf`
- `restaurant_equipment_list_v3.xlsx`

## 上传注意事项

1. **文件大小限制**：单个文件大小建议不超过100MB
2. **编码格式**：文件名请使用UTF-8编码
3. **重复文件**：系统会自动为重复文件名添加时间戳后缀
4. **文件类型识别**：系统会根据文件扩展名自动识别文件类型

## 常见问题及解决方法

### 文件名乱码
- **原因**：文件名使用了非UTF-8编码
- **解决方法**：重命名文件为英文或数字，重新上传

### 文件上传失败
- **原因**：文件过大或网络问题
- **解决方法**：尝试上传较小的文件，或检查网络连接

### 3D模型无法预览
- **原因**：文件格式不支持或文件损坏
- **解决方法**：确保上传的是支持的3D模型格式，且文件完整无损

## 最佳实践

1. **统一命名规范**：为所有项目文件建立统一的命名规范
2. **版本控制**：在文件名中包含版本号，便于追踪文件变更
3. **分类管理**：根据文件类型和项目进行分类管理
4. **定期清理**：定期清理不需要的文件，保持存储空间整洁

遵循以上规范，可以有效避免文件上传乱码问题，确保文件管理的规范性和可靠性。
