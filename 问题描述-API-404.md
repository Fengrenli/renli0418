# 问题描述：Express 5.x `/api/project/:id` 路由返回 404

## 环境
- **Express:** 5.2.1
- **Node.js:** 18.18.0
- **运行方式:** PM2 + tsx (TypeScript 直接执行)
- **服务器:** Alibaba Cloud Linux
- **项目路径:** `/www/wwwroot/renliyesheng/`
- **端口:** 3800

## 问题现象

`/api/project/:id` 路由返回 404，但其他所有 API 路由正常工作。

```bash
# ✅ 正常
curl -s http://127.0.0.1:3800/api/health
# 返回: {"code":200,"msg":"server is running","data":{"status":"ok","dbConnected":true}}

# ✅ 正常
curl -s http://127.0.0.1:3800/api/projects
# 返回: {"code":200,"data":[...项目列表...]}

# ❌ 失败
curl -s http://127.0.0.1:3800/api/project/proj-1775203993102
# 返回: {"code":404,"msg":"API endpoint not found"}
```

## 关键线索

### 1. 路由确实存在
`server.ts` 第 276 行：

```typescript
app.get('/api/project/:id', async (req, res) => {
  try {
    const { id } = req.params;
    const result = await pool.query('SELECT * FROM projects WHERE id = $1', [id]);
    
    if (result.rowCount === 0) {
      return res.status(404).json({ success: false, msg: '项目不存在' });
    }
    // ... 返回项目数据
  } catch (err) {
    // ...
  }
});
```

### 2. 错误消息来源不明
- `"API endpoint not found"` 这个字符串在 server.ts 源码中**不存在**
- `grep -rn "endpoint not found" /www/wwwroot/renliyesheng/ --include="*.ts" --include="*.js"` 返回空
- 注意：如果路由正常匹配但项目不存在，应该返回 `{"success":false,"msg":"项目不存在"}`，而不是 `"API endpoint not found"`

### 3. 路由定义顺序
```
Line 200: app.use(express.static(path.join(__dirname, 'dist')));  // 静态文件
Line 203: app.use('/uploads', express.static(uploadsDir));
Line 206: app.get('/api/test-uploads-dir', ...)   ✅ 工作
Line 222: app.get('/api/projects', ...)            ✅ 工作
Line 276: app.get('/api/project/:id', ...)         ❌ 404
Line 329: app.post('/api/upload', ...)             （未测试）
Line 394: app.post('/api/save-project-detailed', ...)
```

### 4. 之前的操作
- 之前通过宝塔 web 终端多次使用 `sed` 修改 server.ts
- 后来用完整文件替换了 server.ts
- `grep` 确认修复代码已到位
- `pm2 restart all` 已执行多次

## 已排除的可能性
- ❌ **不是 nginx 问题** — 直接 curl localhost:3800 同样 404
- ❌ **不是数据库问题** — 项目在数据库中存在
- ❌ **不是文件缺失** — server.ts 中 grep 确认路由代码存在
- ❌ **不是语法问题** — 其他路由正常工作

## 待排查方向

### 方向 1: PM2 运行的不是最新 server.ts
```bash
pm2 show renliyesheng  # 查看 script path 和 interpreter
pm2 delete renliyesheng  # 完全删除进程
pm2 start server.ts --name renliyesheng --interpreter npx -- tsx  # 全新启动
```

### 方向 2: Express 5.x 路由语法变化
Express 5.x 使用 path-to-regexp v8，路由参数语法可能有变化：
- Express 4: `/api/project/:id` → 正常匹配
- Express 5: 可能需要检查 path-to-regexp v8 的行为

### 方向 3: 检查是否有编译产物被加载
```bash
ls -la /www/wwwroot/renliyesheng/server.js  # 检查是否有 .js 版本
find /www/wwwroot/renliyesheng -name "*.js" -path "*/server*" -not -path "*/node_modules/*"
```

### 方向 4: 之前 sed 操作可能注入了 catch-all 中间件
虽然已用完整文件替换，但需确认：
```bash
# 搜索可能的 catch-all 404 handler
grep -n "app.use\|app.all\|res.status(404)" /www/wwwroot/renliyesheng/server.ts
# 查看文件末尾是否有多余代码
tail -30 /www/wwwroot/renliyesheng/server.ts
```

### 方向 5: 直接最小化测试
创建一个只有问题路由的最小 server 来隔离问题：
```typescript

import express from 'express';
const app = express();
```
