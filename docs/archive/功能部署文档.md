# 功能部署文档

## 文档信息
- **文档名称**：功能部署文档
- **版本**：v1.0
- **创建日期**：2026年3月23日
- **适用范围**：品牌客户专属登录、物流实时追踪API、在线即时通讯功能部署

## 一、功能概述

### 1. 品牌客户专属登录
- **功能描述**：为每个品牌客户提供专属账号，实现数据隔离，让品牌客户只能看到自己名下的项目
- **技术实现**：扩展用户表，添加品牌相关字段，实现基于品牌的权限控制

### 2. 物流实时追踪API
- **功能描述**：接入国际货运API，实现物流信息的自动同步和实时追踪
- **技术实现**：创建物流追踪表，开发物流相关API接口

### 3. 在线即时通讯
- **功能描述**：添加WhatsApp和微信客服悬浮按钮，提供快速沟通渠道
- **技术实现**：创建CustomerSupportButton组件，集成到网站中

## 二、部署步骤

### 1. 服务器配置

#### 1.1 环境要求
- Node.js 16.0+ 
- PostgreSQL 12.0+ 
- Nginx 1.18+ 
- PM2 5.0+ 

#### 1.2 依赖安装
```bash
# 安装项目依赖
npm install

# 安装PM2
npm install -g pm2
```

### 2. 数据库配置

#### 2.1 数据库结构
- **users表**：已扩展添加brand_id和brand_name字段
- **logistics_tracking表**：新增表，用于存储物流追踪信息

#### 2.2 数据库初始化
服务器启动时会自动创建必要的数据库表结构，包括：
- users表（带品牌字段）
- logistics_tracking表

### 3. 配置文件

#### 3.1 环境变量
编辑`.env`文件，添加以下配置：
```
# 服务器配置
SERVER_PORT=3800

# 数据库配置
DATABASE_URL=postgresql://username:password@localhost:5432/database

# 其他配置
NODE_ENV=production
```

### 4. 部署命令

#### 4.1 开发环境
```bash
# 启动开发服务器
npm run dev
```

#### 4.2 生产环境
```bash
# 构建项目
npm run build

# 启动生产服务器
npm run start

# 或使用PM2管理
npm run deploy
```

## 三、功能使用说明

### 1. 品牌客户专属登录

#### 1.1 创建品牌客户账号
1. 登录管理员后台
2. 进入用户管理页面
3. 点击"添加用户"
4. 填写用户名、密码，选择角色为"brand"，并填写品牌ID和品牌名称
5. 点击"保存"

#### 1.2 品牌客户登录
1. 访问网站登录页面
2. 输入品牌客户账号和密码
3. 登录后只能看到自己品牌的项目

### 2. 物流实时追踪API

#### 2.1 创建物流追踪记录
- **API接口**：`POST /api/logistics/create`
- **请求参数**：
  - project_id：项目ID
  - tracking_number：物流单号
  - carrier：承运商
  - status：物流状态
  - tracking_data：详细追踪数据（JSON格式）

#### 2.2 获取物流追踪记录
- **API接口**：`GET /api/logistics/list`
- **查询参数**：
  - project_id：项目ID（可选）

#### 2.3 更新物流追踪状态
- **API接口**：`POST /api/logistics/update`
- **请求参数**：
  - id：追踪ID
  - status：物流状态
  - tracking_data：详细追踪数据（JSON格式）

#### 2.4 删除物流追踪记录
- **API接口**：`POST /api/logistics/delete`
- **请求参数**：
  - id：追踪ID

### 3. 在线即时通讯

#### 3.1 使用方法
1. 网站右下角会显示一个悬浮的客服按钮
2. 点击按钮会展开客服面板
3. 选择联系方式（WhatsApp、微信或电话）
4. 直接与客服进行沟通

#### 3.2 配置说明
- WhatsApp号码：在`CustomerSupportButton.tsx`文件中修改`phoneNumber`变量
- 微信二维码：在`CustomerSupportButton.tsx`文件中修改二维码图片URL
- 电话号码：在`CustomerSupportButton.tsx`文件中修改电话链接

## 四、故障排除

### 1. 品牌客户登录问题
- **问题**：品牌客户登录后看不到项目
  **解决**：检查用户的brand_id是否与项目的client_name匹配

- **问题**：品牌客户无法登录
  **解决**：检查账号密码是否正确，以及用户状态是否为active

### 2. 物流追踪API问题
- **问题**：物流追踪记录创建失败
  **解决**：检查项目ID是否存在，物流单号是否唯一

- **问题**：物流状态更新失败
  **解决**：检查追踪ID是否存在

### 3. 在线即时通讯问题
- **问题**：客服按钮不显示
  **解决**：检查`CustomerSupportButton.tsx`组件是否正确集成到App.tsx中

- **问题**：WhatsApp链接无效
  **解决**：检查电话号码格式是否正确，国际号码需要包含国家代码
