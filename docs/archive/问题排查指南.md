## 🎯 可能的问题和解决方案

### 问题 1：前端构建文件不完整

**症状：** 页面加载但没有任何内容显示

**解决方案：**
```bash
# 在本地重新构建项目
npm run build

# 确认 dist 目录内容
ls -la dist/

# 上传整个 dist 目录到服务器
# 使用 FTP 或 SCP 工具上传
```

### 问题 2：Nginx 配置问题

**症状：** API 请求返回 404 或 502 错误

**解决方案：**
```bash
# 检查 Nginx 配置
nginx -t

# 重新加载 Nginx
nginx -s reload

# 检查 Nginx 是否正在运行
systemctl status nginx
```

**确认 Nginx 配置包含以下内容：**
```nginx
# 后端API接口反向代理
location /api {
    proxy_pass http://127.0.0.1:3800;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

### 问题 3：PM2 进程问题

**症状：** API 请求超时或连接被拒绝

**解决方案：**
```bash
# 检查 PM2 进程状态
pm2 list

# 如果进程没有运行，启动它
pm2 start server.ts --name renliyesheng

# 重启进程
pm2 restart renliyesheng

# 查看进程日志
pm2 logs renliyesheng
```

### 问题 4：端口被占用

**症状：** 后端服务无法启动

**解决方案：**
```bash
# 检查 3800 端口是否被占用
netstat -tlnp | grep 3800

# 如果被占用，杀死进程或修改端口
# 修改 .env 文件中的 SERVER_PORT
```

### 问题 5：环境变量配置错误

**症状：** 数据库连接失败

**解决方案：**
```bash
# 检查服务器上的 .env 文件
cat /www/wwwroot/renliyesheng/.env

# DB_HOST=rm-cn-pj64od8bk00014zo.rwlb.rds.aliyuncs.com
# DB_USER=renli_2026

# DB_DATABASE=renli_company
# DB_PORT=5432
# SERVER_PORT=3800
```

### 问题 6：浏览器缓存问题

**症状：** 修改后仍然显示旧内容

**解决方案：**
1. 在浏览器中按 `Ctrl + Shift + R` 强制刷新
2. 清除浏览器缓存
3. 使用无痕/隐私模式测试

### 问题 7：CORS 跨域问题

**症状：** 控制台显示 CORS 相关错误

**解决方案：**
确认 `server.ts` 中已配置 CORS：
```typescript
app.use(cors());
```

如果还有问题，可以尝试更宽松的配置：
```typescript
app.use(cors({
    origin: '*',
    methods: ['GET', 'POST', 'PUT', 'DELETE'],
    allowedHeaders: ['Content-Type', 'Authorization']
}));
```

## 🆕 新增问题：数字资产（模型）上传后刷新页面消失

### 🔍 问题描述
用户在网页端上传 3D 模型等数字资产后，页面能立即显示。但一旦刷新浏览器，这些资产就消失了。

### 🧠 根本原因
前端在上传文件后，会调用 `onUpdateProjectDetails` 函数，并尝试向 `/api/save-project-detailed` 接口发送保存请求。**但是，后端 `server.ts` 文件中并未实现此接口**，导致所有保存请求均返回 `404 Not Found`。因此，`projects` 表中的 `digital_assets` 字段从未被更新，始终为空。刷新页面时，前端从 `/api/projects` 获取的数据自然不包含任何资产信息。

### 🔧 诊断步骤
1. **打开浏览器开发者工具 (F12)**。
2. 切换到 **Network (网络) 标签页**。
3. **上传一个模型文件**。
4. 在 Network 面板中，**查找名为 `save-project-detailed` 的 POST 请求**。
5. **检查其状态码**：
   - 如果是 `404`，则确认是后端接口缺失。
   - 如果是 `200`，则需进一步检查数据库是否被正确更新。

### 🛠️ 解决方案
在 `server.ts` 文件中**添加缺失的 `/api/save-project-detailed` 接口**。

**接口核心逻辑**：
- 接收前端传来的完整项目数据 `req.body`。
- 提取项目ID `projectId`。
- 构建 SQL `UPDATE` 语句，**确保包含 `digital_assets` 字段**。
- 执行更新并返回成功响应。

**示例代码**：
```typescript
app.post('/api/save-project-detailed', async (req, res) => {
  try {
    const projectData = req.body;
    const projectId = projectData.id;
    // ... [参数验证] ...
    
    // 构建更新字段，必须包含 'digital_assets'
    const fieldsToUpdate = ['name', 'client_name', ..., 'digital_assets'];
    // ... [构建SQL并执行] ...
    
    sendResponse(res, 200, { code: 200, msg: '项目保存成功', success: true });
  } catch (error) {
    console.error('保存项目详细信息时出错:', error);
    sendResponse(res, 500, { code: 500, msg: '服务器内部错误', success: false });
  }
});
```

### ✅ 验证修复
1. 添加接口后，重启服务 (`pm2 restart renliyesheng`)。
2. 重新上传一个模型。
3. 刷新页面，模型应持续显示。
4. 在 Network 面板中确认 `save-project-detailed` 请求返回 `200`。
