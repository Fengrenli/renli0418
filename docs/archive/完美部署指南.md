# 🚀 成都仁力烨升国际贸易有限公司 - 阿里云完美部署指南

本指南将引导您完成网站在阿里云（Aliyun）上的全流程部署，包括 RDS 数据库配置、环境配置及线上发布。

---

## 1. 🗄️ RDS 数据库 (PostgreSQL) 创建与配置

您的项目使用 **PostgreSQL** 数据库。请按照以下步骤在阿里云 RDS 中完成配置。

### 第一步：创建 RDS 实例
1. 登录 [阿里云控制台](https://home.console.aliyun.com/)。
2. 搜索 **RDS**，进入“关系型数据库 RDS”控制台。
3. 点击“创建实例”，选择：
   - **引擎**：PostgreSQL
   - **版本**：15 或更高
   - **系列**：基础版或高可用版（根据预算选择）
   - **网络类型**：VPC（确保与您的 ECS 或 SAE 处于同一 VPC）

### 第二步：配置账号与数据库
1. **创建账号**：在 RDS 实例详情页 -> 账号管理 -> 创建账号（例如 `renli_admin`）。
2. **创建数据库**：在数据库管理 -> 创建数据库（例如 `renli_company`）。
3. **设置白名单**：在“白名单与安全组”中，添加您的 ECS 公网 IP 或 VPC 内网网段（若使用内网连接），确保应用可以访问数据库。

### 第三步：手动初始化表格（可选）
虽然应用启动时会自动尝试创建表，但建议您在 RDS 的 **DMS (数据管理)** 中手动执行以下 SQL 以确保万无一失：

```sql
-- 1. 项目管理表
CREATE TABLE IF NOT EXISTS projects (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL DEFAULT 'Untitled Project',
  client_name TEXT,
  project_type TEXT,
  area TEXT,
  location TEXT,
  city TEXT,
  country TEXT,
  coordinates JSONB,
  status TEXT,
  description TEXT,
  progress INTEGER DEFAULT 0,
  stages JSONB,
  digital_assets JSONB,
  image_url TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 2. 材料库表
CREATE TABLE IF NOT EXISTS materials (
  id SERIAL PRIMARY KEY,
  code TEXT UNIQUE,
  name TEXT,
  image TEXT,
  description TEXT,
  price NUMERIC,
  unit TEXT,
  spec TEXT,
  color TEXT,
  area TEXT,
  hs_code TEXT,
  weight NUMERIC,
  volume NUMERIC,
  create_time TIMESTAMP DEFAULT NOW()
);

-- 3. BOM 项目主表
CREATE TABLE IF NOT EXISTS bom_projects (
  id SERIAL PRIMARY KEY,
  project_name TEXT,
  order_unit TEXT,
  contact TEXT,
  supplier TEXT,
  notes TEXT,
  total_cny NUMERIC,
  total_eur NUMERIC,
  create_time TIMESTAMP DEFAULT NOW()
);

-- 4. BOM 项目明细表
CREATE TABLE IF NOT EXISTS bom_project_items (
  id SERIAL PRIMARY KEY,
  project_id INTEGER REFERENCES bom_projects(id),
  material_id TEXT,
  quantity NUMERIC,
  price_cny NUMERIC,
  price_eur NUMERIC,
  remark TEXT,
  box_no TEXT
);

-- 5. 招商加盟报名表
CREATE TABLE IF NOT EXISTS campaign_application (
  id SERIAL PRIMARY KEY,
  full_name TEXT,
  email TEXT,
  phone TEXT,
  preferred_city TEXT,
  investment_budget TEXT,
  status TEXT DEFAULT 'pending',
  create_time TIMESTAMP DEFAULT NOW()
);
```

## 3. ☁️ 应用部署 (ECS 或 SAE)

推荐使用 **SAE (Serverless 应用引擎)** 或 **ECS (云服务器)**。

### 方案 A：使用 ECS (传统服务器)
1. **环境准备**：
   - 安装 Node.js (v18+)。
   - 安装 PM2：`npm install pm2 -g`。
2. **代码上传**：将代码上传至服务器。
3. **安装依赖**：`npm install`。
4. **构建前端**：`npm run build`。
5. **配置环境变量**：创建 `.env` 文件（参考下文）。
6. **启动应用**：`pm2 start server.ts --interpreter tsx --name renli-app`。

### 方案 B：使用 SAE (推荐，更简单)
1. 在 SAE 控制台创建应用。
2. **代码源**：选择 GitHub/Gitee 或直接上传 ZIP 包。
3. **构建配置**：
   - 构建命令：`npm install && npm run build`
   - 启动命令：`npm run start`
4. **环境变量**：在 SAE 控制台配置下方的环境变量。

## 4. 🔑 环境变量配置 (.env)

在您的服务器或部署平台中设置以下变量：

| 变量名 | 说明 | 示例 |
| :--- | :--- | :--- |
| `NODE_ENV` | 环境模式 | `production` |
| `DB_HOST` | RDS 连接地址 | `rm-xxx.rwlb.rds.aliyuncs.com` |
| `DB_USER` | RDS 账号 | `renli_admin` |
| `DB_PASSWORD` | RDS 密码 | `您的密码` |
| `DB_DATABASE` | 数据库名 | `renli_company` |
| `DB_PORT` | 数据库端口 | `5432` |
| `DB_SSL` | 是否启用 SSL | `false` (内网连接通常设为 false) |
| `GEMINI_API_KEY` | AI 功能密钥 | `您的 Gemini API Key` |

---

## 5. 🌐 域名与 SSL
1. **域名解析**：在阿里云解析 DNS 中，将您的域名指向 ECS 公网 IP 或 SAE 提供的公网访问地址。
2. **SSL 证书**：在阿里云“数字证书管理服务”中申请免费证书，并在负载均衡 (SLB) 或 Nginx 中配置 HTTPS。
