# 成都仁力烨升国际贸易有限公司 - 阿里云部署维护指南

## 一、服务器环境准备

### 1. 阿里云服务器配置
- **推荐配置**：2核4G内存，40GB SSD磁盘，CentOS 7.9操作系统
- **安全组配置**：
  - 开放端口：80（HTTP）、443（HTTPS）、3000（开发服务器）、22（SSH）
  - 允许所有IP访问这些端口

### 2. 服务器初始化
1. **登录服务器**：使用SSH工具（如PuTTY）连接服务器
   - 用户名：root
   - 密码：服务器创建时设置的密码

2. **更新系统**：
   ```bash
   yum update -y
   ```

3. **安装必要软件**：
   ```bash
   yum install -y wget curl git gcc-c++ make
   ```

4. **安装Node.js**：
   ```bash
   curl -sL https://rpm.nodesource.com/setup_16.x | bash -
   yum install -y nodejs
   ```

5. **安装PM2**：
   ```bash
   npm install -g pm2
   ```

6. **安装Nginx**：
   ```bash
   yum install -y nginx
   systemctl enable nginx
   systemctl start nginx
   ```

### 3. Nginx配置
1. **编辑Nginx配置**：
   ```bash
   nano /etc/nginx/conf.d/renliyesheng.conf
   ```

2. **粘贴以下配置**：
   ```nginx
   server {
       listen 80;
       server_name www.renliyesheng.net renliyesheng.net;
       return 301 https://$host$request_uri;
   }

   server {
       listen 443 ssl;
       server_name www.renliyesheng.net renliyesheng.net;
       
       ssl_certificate /etc/nginx/ssl/fullchain.pem;
       ssl_certificate_key /etc/nginx/ssl/privkey.pem;
       ssl_protocols TLSv1.2 TLSv1.3;
       ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-SHA384;
       
       root /www/wwwroot/renliyesheng;
       index index.html;
       
       location / {
           try_files $uri $uri/ /index.html;
       }
       
       location /api {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
       
       location /uploads {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

3. **创建SSL证书目录**：
   ```bash
   mkdir -p /etc/nginx/ssl
   ```

4. **上传SSL证书**：
   - 将SSL证书文件（fullchain.pem和privkey.pem）上传到 `/etc/nginx/ssl` 目录

5. **重启Nginx**：
   ```bash
   systemctl restart nginx
   ```

### 2. 材料文件处理流程

#### 步骤1：准备材料文件
- **材料文件位置**：`C:\Users\Admin\Desktop\material`
- **文件类型**：
  - 图片文件（.jpg, .png等）
  - Excel文件（.xlsx）包含材料信息

#### 步骤2：批量处理材料文件
1. **上传图片文件**：
   - 使用FTP工具将 `C:\Users\Admin\Desktop\material` 中的图片文件上传到服务器的 `/www/wwwroot/renliyesheng/uploads` 目录

2. **处理Excel文件**：
   - 打开Excel文件，确保包含以下字段：
     - code：材料编码
     - name：材料名称
     - image：图片URL（相对路径，如 `/uploads/图片文件名.jpg`）
     - price：价格
     - unit：单位
     - spec：规格
     - color：颜色
     - area：面积
     - hs_code：HS编码

#### 步骤3：导入材料数据到数据库
1. **使用导入脚本**：
   ```bash
   cd /www/wwwroot/renliyesheng
   node import_materials_from_csv.js
   ```

2. **验证导入结果**：
   - 登录管理后台查看材料列表
   - 确认所有材料都已正确导入

#### 步骤4：关联材料到项目
1. **在管理后台**：
   - 进入项目详情页面
   - 点击"上传本地文档"按钮
   - 选择材料文件进行上传
   - 系统会自动关联材料到项目

### 3. 材料数据补全流程
由于材料信息需要分两次上传才能补全，推荐以下流程：

1. **第一次上传**：上传所有图片文件
   - 将所有材料图片上传到服务器
   - 记录每个图片的相对路径

2. **整理Excel文件**：
   - 在Excel中填写所有材料信息
   - 在image字段中填写对应的图片相对路径

3. **第二次上传**：导入Excel数据
   - 运行导入脚本将Excel数据导入数据库
   - 系统会自动关联图片和材料信息
